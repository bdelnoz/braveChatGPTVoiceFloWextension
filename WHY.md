# Why ChatGPT Voice Flow Exists

ChatGPT Voice Flow exists because ChatGPT Web voice workflow still requires too many manual actions.

The missing native feature is simple:

AutoSend for voice transcription.

From that missing button, the workflow expands naturally:

voice recording → transcription → send → answer → Read aloud → continue.

## Why not native conversation mode?

The native conversation mode does not behave like the normal ChatGPT Web text flow.

This extension keeps the normal ChatGPT Web interaction model and adds voice automation around it.

## Why local?

The extension is local-first.

No external service is required.

No cloud backend is required.

No account capture is required.

It is browser automation for the user’s own workflow.

## Design principle

Small working tool first.

No documentation swamp.

No corporate varnish.

Make the daily workflow faster.
